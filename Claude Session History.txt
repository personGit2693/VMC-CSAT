● Now I have a complete picture. Here are the specific performance issues I
  found:

  1. innerHTML += in 4 View files — browser serializes the entire existing DOM
  to string, concatenates, and re-parses everything. insertAdjacentHTML appends
  without touching existing nodes.
  2. String concatenation += in loops across all table/list Components — creates
   O(N²) intermediate string objects. array.push + .join('') is faster.
  3. .toFixed(0) called 2–6× per row (condition + display string) — should
  compute once per row.
  4. Redundant else if(x != "") after if(x == "") — dead logic in 4 View files.
  5. Intermediate let variable then return in loaders — no need for the
  variable.

  Applying all fixes now.
